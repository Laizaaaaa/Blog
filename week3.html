Week 3: Development Setup

March 8-14 2026

6 min read

Starting Development on Cynthia Curry POS

Week three was the start of the real development work on the Cynthia Curry POS system. On Monday (March 9th), our team kicked off with Gwyn setting up the Laravel project and creating the GitHub repository for version control. While Gwyn and Grace started working on the UI/front-end, I reviewed the database table structures and made some adjustments based on our design discussions.

Creating the Database Seeders

On Tuesday (March 10th), I drafted seeders for the Products, Categories, and Sub-categories tables. These seeders would help us populate the database with initial data from the original CCPOS system. This was important for testing and making sure our new system had all the products ready to go.

Migrations and Data Organization

On Wednesday (March 11th), I created migrations for the Products, Categories, and Sub-categories tables. As our team reviewed the database again, we made some changes to the table structure to make them better. I then finalized the product seeder and organized all the products by category – this was different from the original system where products were mixed together, making it easier to update them later if needed. I also copied all the product images from the original CCPOS files to our Laravel project and uploaded them to GitHub so everyone could access them.

Working on the Menu Search Feature

On Thursday (March 12th), I worked on making the search feature in the menu work properly. I found and fixed some bugs in the search feature and added a "no results found" message so users would know if their search didn't find anything.

Fixing Final Issues

On Friday (March 13th), I spent the day fixing remaining problems. I renamed some migration files so they would run in the right order when setting up the database. I also fixed the payment details calculation based on how the staff had calculated it in the original system to make sure everything was accurate.

Key Learnings from Week 3

  • How to set up and organize a Laravel project properly
  • Creating database seeders to populate test data
  • Working with database migrations and making changes when needed
  • Organizing data in a way that makes it easy to update later
  • Building features like search that users need
  • Fixing bugs and making sure calculations are correct

Week 3 Reflection

This week felt good because I was building real parts of the system that people would use. Setting up the database, organizing the data, and making features like search work taught me a lot. I was working on actual code that mattered, not just practice exercises. By the end of the week, the foundation of the system was in place and we were ready to continue building more features.
