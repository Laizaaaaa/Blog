Week 6: Continuing POS Development

March 30-31 2026

4 min read

Polishing the POS Features

This short week focused on refining the POS system. On Monday (March 30th), I made the order status filter work so kitchen staff can see orders organized by their status. On Tuesday (March 31st), I connected the tables to the Order model so the system knows which table each order came from, then fixed hover issues in the filter to make it feel smooth.

Making It Better and Flexible

I also removed the 'w/' abbreviation from all product names based on the owner's feedback. More importantly, I changed the room filter from being hard-coded to dynamic – meaning new rooms can be added from the database without changing the code. This makes the system flexible for when the restaurant grows.

Key Learnings from Week 6

  • Making filters that help organize information
  • Connecting database relationships properly
  • Making code flexible for future growth

Week 6 Reflection

Even though this was a short week, I made meaningful improvements to the POS system. Refining features and making the code flexible showed me that good software is built with both current needs and future growth in mind.
