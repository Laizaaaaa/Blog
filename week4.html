Week 4: Refining Features

March 15-21 2026

7 min read

Fixing Data Issues in the System

Week four started with finding and fixing problems with the data. On Monday (March 16th), I noticed that the product categories were jumbled because of changes in the category seeder. I fixed the product seeder to match the new category structure. I also changed the timezone settings in the app to match our location, and made the customer name field optional since not all orders need a customer name.

Improving the System Features

On Monday, I also moved some order-related functions to a new controller file to make the code easier to read and understand. Controllers help organize code by grouping related functions together. Some of these functions still had default values that would later be replaced with real data from users.

Working on User Interface and Authentication

On Tuesday (March 17th), I fixed the error message system so that when someone enters the wrong PIN, the error shows up in the PIN popup instead of on the page. I also created seeders for user roles, accounts, and users so we could test the system properly. I fixed an issue with the search feature where the cursor would disappear after showing results. Instead of using JavaScript code on the webpage to filter categories, I switched to using the server (the backend) to do the filtering, which works better.

Adding Customer Features

On Wednesday (March 18th), I added a feature that suggests customer names when you're filling out an order. I also fixed some bugs that came up while testing. The team got together to discuss how the project was going and gave some suggestions on small changes to improve things.

Fixing and Documenting the Code

On Thursday (March 19th), I fixed the MenuController and added comments throughout the code to explain what each part does.

Key Learnings from Week 4

  • How to find and fix problems with data in the database
  • Organizing code into separate files makes it easier to work with
  • Making small features like error messages work better improves the user experience
  • Using comments in code helps others understand what the code does
  • Sometimes the server can do work better than the webpage
  • Testing features before they're finished helps find problems early

Week 4 Reflection

This week was about making the system work better and making the code cleaner. I fixed real problems that were making the system confusing, and I organized the code in a way that makes it easier to keep working on. By the end of the week, the POS system was starting to look and work like a real product that the staff could use.
